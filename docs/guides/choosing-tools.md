# When You Need Dedicated Tools

> An honest look at when plain-text specs are enough, when dedicated tooling starts to pay off, and what to look for when you're ready.

*This guide is coming soon. In the meantime, here's what it will cover:*

- **When text files are enough**: solo projects, small teams, greenfield work with clear domains
- **When tools start to help**: cross-team coordination, large domains, compliance requirements, brownfield codebases
- **What to look for**: the capabilities that matter (executability, traceability, multi-audience access, agent guidance)
- **The current landscape**: an overview of what's available today, linked to the [Tool Landscape](/landscape/)
- **DIY vs. buy**: what's hard to build yourself and what's worth the dependency
