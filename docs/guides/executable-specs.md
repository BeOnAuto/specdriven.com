# Making Specs Executable

> Specifications that verify themselves. How to connect your specs to BDD, TDD, and modern test frameworks so they stay true as your code evolves.

*This guide is coming soon. In the meantime, here's what it will cover:*

- **What executable specs are**: specs that run as tests, not documents that rot in a wiki
- **BDD with Cucumber/SpecFlow**: connecting Gherkin scenarios to step definitions
- **TDD-by-AI**: writing specs first, letting AI generate both implementation and tests
- **Framework-agnostic patterns**: using Jest, pytest, RSpec, or any test runner
- **The verification loop**: how executable specs catch drift between intent and implementation
- **100% coverage through specification**: why TDD-by-AI makes full coverage achievable by definition
