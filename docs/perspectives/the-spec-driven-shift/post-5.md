# Code Was Always a Hindrance. I Just Didn't Know It Yet.

Before AI, a skilled developer could build a house in a week. Now with AI, an amateur can build a house in a day. Me? I can build a skyscraper in a day. And soon, a moon base.

Everyone keeps saying AI levels the playing field. And yeah, it does. For houses. It's dangerously wrong for skyscrapers.

What separates the amateur house from the expert skyscraper isn't the code. AI writes code for both. What separates them is the design. The architecture. The understanding of which constraints matter and why. The ability to think about information flow, about coupling, about what happens when this thing needs to change in six months.

I barely touch code anymore. I'm more productive than I've ever been. And I know that sounds like every other AI bro on LinkedIn saying the same thing, which is why I want to be precise about what I mean.

I didn't stop coding because AI made it easy. I stopped because I realized the code was never the point. I loved coding. I still love the problem-solving part of it. The satisfaction of an elegant algorithm, a system that handles edge cases gracefully. But what I actually loved all along was the design. The solving. The architecture. The code was the mechanical translation of designs into something a computer could execute. And that translation ate enormous amounts of time that could have been spent on harder problems and better designs.

Now that translation is nearly instant. What remains is the thinking.

So when someone says AI will replace developers, I'd be more specific. AI will replace people who only write code. It'll make people who design systems absurdly powerful.

The question has changed. It's not "can you get AI to produce code?" Obviously it can. It's "can you specify what it should build?" Can you think about the architecture, the trade-offs, the domain model, the edge cases? Can you design?

If you can, you just went from building houses to building skyscrapers. If you can't, you're building houses faster. They'll just fall down sooner.

Next time you sit down to build something with AI, notice where you spend your time. If most of it is on prompts and code, ask yourself: did anyone design this first?
