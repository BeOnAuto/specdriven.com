# Specs Are the Next Programming Languages

![Specs Are the Next Programming Languages](/images/perspectives/specs-are-the-next-programming-languages.png)

Here's what I think happens in the next few years. We end up with an ecosystem of specification dialects as varied as programming languages.

Look at the history of abstraction. Wired circuits. Machine code. Assembly. High-level languages. Frameworks. Each generation abstracted the previous one. Each gave people a higher-order way to say what they wanted.

AI is the next abstraction layer. And it needs its own higher-order languages. Not programming languages that tell machines what to do step by step. Specification languages that tell AI what to achieve.

I call them spec dialects.

Why dialects and not one universal language? Because humans already use different ones instinctively. If I'm designing a building, I think in meters and load capacities and material grades. If I'm designing a boat, hull shapes and displacement. If I'm designing e-commerce software, customers and orders and payments and fulfillment. Different domains need different vocabularies.

My work on Narrative-Driven Development is a spec dialect for line-of-business applications. It uses time-based modeling, domain linguistics, information architecture. But I wouldn't use it to specify a compression engine. Or a game physics system. Or a flight controller. Those need their own dialects.

Right now most people are using plain English as their spec dialect. Natural language prompts. That's like programming in untyped pseudocode. Fine for small things. Falls apart at scale because natural language is inherently ambiguous. "Handle the error" means something different to every person who reads it and every AI that interprets it.

Some of the dialects that emerge will be general purpose, like Python is for programming. Others will be domain-specific, like MATLAB for math. The teams that figure out their dialect first will communicate with AI at higher bandwidth. More precision. Less ambiguity. Better output.

We're at the assembly language stage of specification right now. High-level spec languages are coming.
