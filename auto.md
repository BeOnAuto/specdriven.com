# Auto Engineer

## Put Your Specs on Auto

Auto Engineer is the spec-driven platform for building software. It takes your specifications and produces production-ready, fully-tested code — not through "prompt and pray," but through a deterministic pipeline with feedback loops and quality checks at every stage.

## The Polaroid vs. the SLR

Many tools today let you go from prompt to app. They're the Polaroid camera of software development — quick snapshot, looks nice enough, but the colors aren't quite right, it won't scale to billboard size, and you're stuck with what you get.

Auto Engineer is the SLR. Every setting is customizable. The system is modular. The output is professional-grade. And just like a professional photographer, you start with knowing what shot you want — the specification — before pressing the shutter.

## How It Works

**Tell the story. Auto writes the code.**

1. **Define behavior** using Narrative-Driven Development — a TypeScript DSL with Given/When/Then patterns that capture your domain's rules in plain language.
2. **Auto generates** a domain model, GraphQL schema, event-sourced server, and React frontend — all derived directly from your specifications.
3. **AI implements** the details with self-correcting feedback loops that catch errors before they reach you.
4. **Quality checks** enforce 100% test coverage, type safety, and linting — automatically, on every run.

This isn't vibe coding. This is AI software development made deterministic.

## The Pipeline

```
Narratives → Domain Model → Server Scaffold → Frontend Scaffold → AI Implementation → Quality Checks → Production Code
```

Each stage validates against your specifications. If something doesn't match, the pipeline corrects itself — up to 5 attempts with error feedback before escalating. Your specifications are the source of truth at every step.
